Yes. This is actually a common practice in air cargo logistics. Many airlines and ground handlers use **barcode or QR code scanning** to retrieve shipment information from a mobile device instead of scanning a paper MAWB, which can get folded, scratched, or damaged.

For a **MAWB (Master Air Waybill)**, I recommend using a **Code 128 barcode** because:

* It's the industry standard for logistics and shipping.
* It supports alphanumeric MAWB numbers (e.g., `176-12345675`).
* Most handheld scanners and mobile barcode scanner apps can read it.
* It's more appropriate than a QR code if the airline's scanners already expect a linear barcode.

You can also include a QR code as a secondary option for mobile users, but the primary barcode should be **Code 128**.
