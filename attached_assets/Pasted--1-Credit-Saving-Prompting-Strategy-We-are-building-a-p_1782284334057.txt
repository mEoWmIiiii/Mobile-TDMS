# Product Requirement Document (PRD)

## Project Overview & Objectives

### Product Name
TDMS Mobile (Cross-Platform Fleet & Logistics Management App)

### Target Audience
* **Truck Drivers:** On-the-road field operators requiring high-contrast, frictionless, fast-loading interfaces for high-vibration and outdoor environments.
* **Logistics Dispatchers / Supervisors:** Mobile-first managers monitoring systemic operational health, tracking active routes, and analyzing exceptions on the move.

### The Core Problem
The current logistics workflow relies on manual paper logging and chaotic messaging groups (e.g., Viber chats) for tracking updates. This creates critical operational blind spots, reporting delays, higher administrative overhead, and structural data fragmentations.

### Executive Solution & Value Proposition
An automated, responsive mobile ecosystem built for iOS and Android. LogiTrack converts physical touchpoints into high-scannability digital forms while replacing manual text communication with hardware-level GPS tracking and real-time UI state machinery.

## Technical Architecture & Real-Time Telematics

### Telematics Features
* **Live Map Canvas Engine:** Embed a real-time vector map tracking ongoing asset location coordinates through hardware background GPS streams.
* **GPS Telematics Verification Banner:** A sticky, green, persistent validation bar anchored at the top layout stating: `[✔] GPS Provider Integrated & Active`.
* **State Automation Engine:** The application must automatically change vehicle indicators and map markers without driver intervention:

| System Trigger Condition | UI State Asset Icon | Trip Progress Bar State |
| --- | --- | --- |
| Speed > 5 km/h outside geofence | Active Moving Truck | Incremental % progression |
| Device inside Terminal geofence | Docked Warehouse Icon | Filled Anchor Milestone |
| Speed = 0 km/h outside geofence for >15 mins | Pulsing Amber/Red Alert Ring | Paused/Warning Flag State |

## Design System & Interactivity Constraints
* **Touch Targets:** Every button, checkbox, and dropdown selector must feature a minimum clickable target surface area of 48px x 48px to guarantee driver ergonomics on bumpy roads.
* **Typography Specs:** Utilize high-contrast sans-serif typefaces (e.g., Inter, SF Pro). Apply bold or semi-bold weights exclusively onto key operational references (`Job No`, `MAWB`, `Plate No`, `1300H`) to optimize legibility under direct sunlight.
* **Color Semantics:** Use a neutral background color paired with strict functional semantic indicators: Green for validated active integrations / On-schedule status, and Amber / Red for route delays or missing documentation parameters.
* **Animation Bounds:** All transitions—including accordion components, dashboard chart expansions, and header mutations—must use explicit ease-in-out physics frameworks running at 60 fps to ensure high perceived system responsiveness.
